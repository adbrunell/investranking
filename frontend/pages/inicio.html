Como os dados são organizados

01

Coleta Automática

Scripts ETL rodam diariamente buscando dados de B3 (cotações), CVM (informes mensais), FNET (documentos e rendimentos) e StatusInvest (proventos). Cada fonte tem seu próprio pipeline de extração e tratamento.

02

Normalização

Os dados brutos são limpos, padronizados e organizados em tabelas relacionais. Cada fundo é identificado pelo CNPJ, garantindo consistência entre fontes distintas.

03

Cálculo dos Indicadores

O ranking é calculado em uma materialized view no banco de dados, combinando liquidez, P/VP, dividend yield e atratividade em uma nota única. Os pesos são ajustáveis pelo usuário.

04

Atualização Contínua

Os scripts são orquestrados pelo run_all.ps1, que gerencia intervalos de atualização: dados diários (cotações, FNET) e dados mensais (CVM) são processados em schedules diferentes para garantir frescor da informação.
